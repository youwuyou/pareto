## What we built

### CFD

2D lid-driven cavity flow (incompressible Navier-Stokes) via Chorin's projection method at N=8, 100 time steps, Re=100. Challenge 1 uses the reference stencil solver. Challenge 2 replaces all stencil ops with native kernel matrices (`grid_laplacian`, `grid_gradient`, `grid_divergence`) — results match to machine-epsilon precision with a smaller IR. Bonus: JAX finite-difference Taylor-Green vortex solver in 3D (N=32³, Re=1600) with FFT pressure projection and semi-Lagrangian advection.

### NMR

Full NMR pipeline for anti-3,4-difluoroheptane (C₇H₁₄F₂, 23 atoms): ab initio ¹H/¹⁹F shieldings via RHF/STO-3G + CPHF (`shieldings_module`), then ¹⁹F spectrum simulation via `ops.expv` time evolution of the 16-spin Hamiltonian. Validated end-to-end on H₂O (7 basis functions); preflight analysis on the full molecule (59 basis functions).

## Why this point on the Pareto frontier

**CFD:** Gateway recommended option selected — native kernel IR enables hardware dispatch to matrix/sparse accelerators not reachable by scalar stencil graphs.

**NMR:** `cpu+gpu` selected — delivers 21–30× speedup over `cpu-only` at zero error and negligible carbon (~0.3 g CO₂). QPU options at larger spin dimensions show 10–73% error and ~103 kg CO₂, making them unviable at current scale.

## What we'd do with more time

### CFD

> The following content is the same as the one in the notebook for the CFD part, written by human.

- Item 1: Quantum Solver for Poisson LSE

In our pipeline, we implement traditional simulation procedures.  and it would be very interesting to actually use quantum algorithms to resolve the Poisson equation (*the bottleneck*). For this, we would want to try our the Harrow–Hassidim–Lloyd (HHL) algorithm for the 2D systems that is Hermitian, a highly specialized algorithm for the FTQC Hybrid device, yet enjoy a logarithmic complexity (in contrast to LSE $O(N^{3})$ system solve used by direct solver).

We may not be able to apply this algorithm for the NS-equations to fall into the turbulent regime, as when advective terms dominate, the system is commonly hyperbolic instead of elliptic, it is not the case where HHL is applicable. Perhaps for such case, an operator splitting argument would be possible to separately solve the elliptic and hyperbolic part, and allow us to still leverage the advantages of the HHL algorithm. 


### NMR

- **Phosphorus cluster (34 spins):** A phosphorus cluster with 7×³¹P + 27×¹H has exactly 34 active spins — matching the qsim-statevec simulator limit (34 qubits). At dim = 2³⁴ ≈ 17 billion, dense classical simulation is infeasible, MPS/tensor-network methods are borderline, and Trotterized quantum time evolution becomes the native approach. We prepared the NMR parameters, extracted the SPINACH ground truth, and built the Hamiltonian construction using `ops.two_body`/`ops.embed`. The preflight analysis at this scale would show whether the platform routes to the quantum simulator or QPU — the exact classical–quantum crossover point for NMR.

- **Execute the full pipeline end-to-end:** The compiled backend currently caps at ~10 basis functions for chemistry and dim=32 for `expv`. With the transport-error fix, the difluoroheptane shieldings (59 basis, 752K tu on `cpu+gpu`) and 16-spin spectrum simulation would run. J-couplings are blocked by a separate gateway bug (transpose error on compiled, `qreset`/`cnot` operand errors on other backends).

- **Larger basis sets:** Upgrading from STO-3G (59 basis) to 6-31G (109 basis) would improve shielding accuracy and let us compare computed chemical shifts directly against the experimental values from the difluoroheptane paper.
